### Task 3: Fix undo-delete toast click handler

**Files:**
- Modify: `src/components/Toast.svelte:10`

The current code uses `{onclick}` shorthand on the button. When `onclick` is `undefined` at tap time (e.g. if the 3-second auto-dismiss timer fires just before the tap), the tap silently does nothing. Wrapping in an arrow function ensures `onclick?.()` handles the `undefined` case safely.

- [ ] **Step 1: Replace `{onclick}` with arrow-function wrapper**

In `src/components/Toast.svelte`, line 10, change:
```svelte
<!-- before -->
<button class="toast" class:clickable={!!onclick} {onclick}>

<!-- after -->
<button class="toast" class:clickable={!!onclick} onclick={() => onclick?.()}>
```

- [ ] **Step 2: Type-check**

Run: `npm run check`
Expected: no errors

- [ ] **Step 3: Manual verification**

Run: `npm run dev`
1. Open the app, open a setlist with songs.
2. Delete a song — the toast appears with indigo text "Song deleted - tap to undo".
3. Tap the toast within 3 seconds — the song reappears in the list.
4. Delete another song, wait 3+ seconds for auto-dismiss — confirm no errors.

- [ ] **Step 4: Commit**

```bash

git commit -m "fix: wrap Toast onclick in arrow function to prevent silent tap failures"
```

## Chunk 2: Performance Screen Layout

### Task 4: Restructure performance screen layout

**Files:**
- Modify: `src/components/PerformanceScreen.svelte`

This task restructures the flex layout so PREV/PAUSE appear above the BPM ring, NEXT is a standalone child at the bottom. The `.controls` div (which previously held all three buttons) is renamed `.secondary-controls` and holds only PREV and PAUSE. NEXT becomes a direct child of `.screen`.

**Size changes:**
- BPM ring: `min(72vw, 280px)` → `min(72vw, 252px)` and `max-width/max-height: 280px` → `252px`
- NEXT button: `96px` → `120px` (width and height)
- `.screen` bottom padding: `36px` → `calc(36px + env(safe-area-inset-bottom, 0px))`

- [ ] **Step 1: Replace the entire `.screen` template block (lines 72–126)**

The current order is: `.top-row` → `.song-info` → `.bpm-ring` → `.controls` (all three buttons). The new order is: `.top-row` → `.song-info` → `.secondary-controls` (PREV + PAUSE only) → `.bpm-ring` → standalone NEXT button.

Replace lines 72–126 wholesale with:

```svelte
<div class="screen">
  <div class="top-row">
    {#if isTTSAvailable()}
      <button
        class="tts-btn"
        class:is-on={$settingsStore.announceSongName}
        onclick={handleToggleTts}
        aria-label={$settingsStore.announceSongName ? 'Disable TTS' : 'Enable TTS'}
      >
        🔊
      </button>
    {/if}

    <button class="exit-btn" onclick={handleExit} aria-label="Exit performance">✕</button>
  </div>

  <div class="song-info">
    <div class="position" class:is-paused={$performanceStore.paused}>
      {$performanceStore.songIndex + 1} / {$performanceStore.totalSongs}
      {#if $performanceStore.paused}&nbsp;· PAUSED{/if}
    </div>
    <div class="song-name">{$performanceStore.currentSong?.name ?? ''}</div>
  </div>

  <div class="secondary-controls">
    <button class="prev-btn" onclick={() => performanceStore.prev()} aria-label="Previous song">
      <span class="btn-icon">◀◀</span>
      <span class="btn-label">PREV</span>
    </button>

    <button
      class="pause-btn"
      class:is-paused={$performanceStore.paused}
      onclick={handlePauseResume}
      aria-label={$performanceStore.running ? 'Pause' : 'Resume'}

      <span class="btn-icon">{$performanceStore.running ? '⏸' : '▶'}</span>
      <span class="btn-label">{$performanceStore.running ? 'PAUSE' : 'RESUME'}</span>
    </button>
  </div>

  <div
    class="bpm-ring"
    class:on-beat={beatActive && $performanceStore.running}
    class:is-paused={$performanceStore.paused}
  >
    <div class="bpm-number">{$performanceStore.currentSong?.bpm ?? '--'}</div>
    <div class="bpm-label">BPM</div>
  </div>

  <button class="next-btn" onclick={() => performanceStore.next()} aria-label="Next song">
    <span class="btn-icon">▶▶</span>
    <span class="btn-label">NEXT</span>
  </button>
</div>
```
